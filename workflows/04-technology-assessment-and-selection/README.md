# Phase 3: Technology Assessment & Selection

## Phase Objective
Help clients systematically evaluate technology solutions or select vendors, reduce procurement risk, and ensure return on technology investment.

## Complete Workflow
```
Requirements Definition → Market Scan → RFP/RFI Preparation & Issuance → Vendor Demo Evaluation → PoC Validation → Decision Recommendation
```

## Prerequisites
- Clear list of technology/business requirements
- Project budget range (at least order-of-magnitude)
- Decision process and key decision-makers identified

## Phase Deliverables
| Deliverable | Description | Delivered To |
|--------|------|---------|
| Technical Due Diligence Report | Technology health assessment of target company/system | Investors/Management |
| RFP/RFI Documents | Standardized request for proposal | Potential Vendors |
| Vendor Scorecard | Weighted multi-dimensional scoring matrix | Selection Decision Committee |
| Selection Recommendation Report | Includes analysis/decision/risk/negotiation advice | Final Decision-Maker |

## Key Success Factors
- **Requirements first, technology second**: Going to vendors without knowing what you need guarantees you'll be led astray
- **You write the demo script, not the vendor**: Design demo scripts based on "must-win scenarios" — eliminate flashy demos
- **Reference calls are mandatory**: At least 3 peer-client references, using a structured question list prepared in advance
- **TCO must include exit costs**: The cost of migrating to a new system/vendor is part of TCO
