# Phase 5: Change Management & Organizational Design

## Phase Objective
Ensure that technology solutions can be successfully adopted within the organization. Even the best technical solution will fail if people don't accept it and the organization doesn't support it. This is a cross-cutting support capability spanning all phases.

## Use Cases
- Large IT projects require a change management plan before launch
- IT organization restructuring / transformation
- New system go-live requires user adoption strategy
- Digital transformation reveals organizational bottlenecks

## Prerequisites
- Technology/strategy proposals have reached preliminary conclusions
- Understanding of the client's current organizational state (available from the diagnostic report)
- Key sources of change resistance have been identified

## Phase Deliverables
| Deliverable | Description | Delivered To |
|--------|------|---------|
| Change Management Plan | Includes: stakeholder analysis / change impact assessment / communication plan / training plan / resistance management | CIO / Change Lead |
| IT Organizational Design Proposal | Target org structure / role definitions / capability model / talent roadmap | CIO/CHRO |
| Change Readiness Assessment | 5-dimension scoring + improvement recommendations | Project Sponsor |

## Key Success Factors
- **Change management is not a PowerPoint deck — it's a daily practice**: Every project meeting should have a change management agenda item
- **Find the right Change Champions**: It's not about title — it's about influence
- **Use Kotter + ADKAR together**: Kotter's 8 Steps at the organizational level, ADKAR at the individual level
- **Measure adoption rate, not just go-live rate**: System go-live ≠ user adoption ≠ business value
