# Phase 1: Project Initiation & Diagnosis

## Phase Objective
After signing, rapidly assemble the team, deeply understand the client's current state, produce a diagnostic report, and lay the foundation for subsequent strategy design.

## Complete Workflow
```
Project Charter & Team Formation → Stakeholder Interviews → IT Current-State Diagnosis (Technology/Process/Organization/Data) → Diagnostic Report Writing & Presentation
```

## Prerequisites
- SOW has been signed
- Client has designated project liaison and key contacts
- Access to foundational documents and data has been obtained

## Phase Deliverables
| Deliverable | Description | Delivered To |
|--------|------|---------|
| Project Charter | Formal document covering project objectives/scope/team/timeline/governance | Both Teams |
| Interview Records | Structured stakeholder interview documentation | Internal |
| IT Current-State Diagnostic Report | Includes maturity scoring / pain point analysis / root causes / quick-win opportunities | Client Executive Leadership |

## Key Success Factors
- **Build credibility in the first week**: Deliver the project charter quickly to demonstrate execution capability
- **CEO/CIO interviews are mandatory**: Without genuine executive perspectives, the diagnosis lacks substance
- **Look beyond technology**: Diagnose process/organization/data/culture simultaneously — technology is only one of four dimensions
- **Diagnostic report must include quick-win opportunities**: Don't just list problems — include 1–3 actions that can be taken immediately
